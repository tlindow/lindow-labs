# Build Logs

Personal build logs tracking the evolution of this site and related projects. Each entry captures reflections, decisions, progress, and context that will be useful when building into future product spaces or services.

## Structure

Logs are organized chronologically as individual Markdown files:

```
logs/
├── README.md
├── 2025-06-XX-initial-site-launch.md
├── 2025-XX-XX-<topic>.md
└── ...
```

## Entry format

Each log entry follows this template:

```markdown
# <Title>

**Date:** YYYY-MM-DD
**Tags:** #tag1 #tag2 #tag3

## Context
What's happening, what prompted this entry.

## Reflections
Thoughts, observations, lessons learned.

## Decisions
Key decisions made and the reasoning behind them.

## Next
What comes next, open questions, future considerations.
```

## Purpose

These logs serve as a personal knowledge base — a record of the journey that can be referenced by future agents, collaborators, or the author when building into new product spaces, services, or features. They provide continuity and context across sessions and time.
